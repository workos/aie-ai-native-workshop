Loops & goals.

Stop babysitting prompts. Hand over a goal and walk away.

Goals

“Done” is a checklist, not a vibe.

✓Reproduce the bug with a failing testThe agent proves the problem exists before touching anything.
✓Implement the fixSmallest change that makes the test pass.
✓Lint · typecheck · full test suite greenNo box is checked on a hunch.
✓Open a PR with the plan in the bodyEvidence, not a vibe.

The issue ships a checklist; the agent isn't done until every box is checked.

/loop — self-paced, recurring work

Goal vs. loop.

Goal / checklist

One bounded job.

Known steps, a clear definition of done. The agent runs the checklist and stops.

Best when you can name every step up front.

→
Loop

Unknown-size or recurring.

/loop keeps going — triaging a backlog, sweeping a codebase, polling for new work.

Best when "done" is a condition, not a count.

Footnote: run parallel agents on git worktrees — isolated checkouts so loops never collide.

Verification gates.

Trust the gates — not the model's confidence.

The cheapest gate

A hook that lints, typechecks, and tests — every change.

lint
typecheck
tests
build

The agent can't hand you code that doesn't build. The hook runs on every change — red turns green, or the work isn't done.

The adversarial gate

Fan out to Codex for a second opinion.

On risky changes, spawn an independent review — then fix everything it finds.

$ codex exec "adversarially review this diff"
# second model, fresh eyes, no stake in being right
found 3 issues · fix all · re-run gates

Two models disagreeing is cheaper than one model being confidently wrong.

Definition of done

Operators trust the gates — not the model's confidence.
